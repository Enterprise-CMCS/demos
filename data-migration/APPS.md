#### Observability

- **Logs**: Rich console output to stderr (operator-facing), structured gate files in `state/`
- **Audit**: `reports/runs/parity_*.md` generated per parity run, `reports/runs/pgloader_*.log` per load
- **Alerts**: None (operator-attended process)

#### Key Operational Notes

- Phases are **gated and idempotent** — each requires prior phase's `state/<phase>.ok`
- All SQL is idempotent (truncate-then-insert)
- FKs added AFTER data load as `NOT VALID`, then validated
- Triggers added LAST to avoid firing during build
- `make rebuild` runs the full pipeline: init → ddl → load_full → seeds → crosswalks → id_maps → build → history → constraints → parity
- Cutover phases (in order): preflight → freeze → delta → build → history → constraints → parity → flip → smoke → decom
- `uv run migrate resume` runs all remaining phases from current gate
- **Ownership boundary:** the target schema (`demos_app`), its tables/FKs/indexes/sequences, lookup seeds, `*_history` tables, history-capture triggers, functions, views, grants (`demos_read`/`demos_write`/`demos_delete`), and cron are **owned by the DEMOS app** (Prisma migrations + `server/src/sql/*` via `refreshDbObjects.ts`). This repo *populates* that schema from MySQL; it does not author it. See `reports/narrative/ownership-remediation-plan.md`.
