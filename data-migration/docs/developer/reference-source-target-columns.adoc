= Reference: source/target column map

The column-level join between MySQL PMDA and DEMOS PostgreSQL.
Useful when authoring or editing transforms in `sql/10_stg/*` and
`sql/20_app/*` -- every column pair lists the transform that must
implement the type bridge.

The fragments below are regenerated by
`docs/tools/source_target_columns_to_adoc.py`; run `make column-map`
from `docs/` after editing `reports/source_target_columns.csv`. Both
schemas are read from their authoritative source of truth, not a
hand-drawn diagram: the MySQL PMDA side from the live
information_schema snapshot at `reports/schema_snapshot/columns.csv`,
and the DEMOS side from the pinned Prisma DDL (replayed through the
same migration folding as the data dictionary, so `ADD COLUMN`
migrations such as `demonstration.medicaid_id` / `demonstration.chip_id`
are present). A schema diagram can still be passed as an override via
`--mysql-mmd` / `--demos-mmd` (used by the tests). While the SME completes the column map, the
`make column-map` target passes `--allow-uncovered-tables` so the
build does not fail on tables that have not been mapped yet -- they
are surfaced as a count and a collapsible list inside the coverage
fragment. The SME view of the same data lives at
xref:../sme/reference-source-target-columns.adoc[the SME column
map].

== Transform vocabulary

The `transform` column is from a controlled vocabulary; the
generator rejects unknown values. Each value implies where the
transform is implemented:

[cols="2,5"]
|===
| Transform | Where implemented

| `copy`
| Direct column copy (with the type cast pgloader assigns by
  default). No semantic transform.

| `cast`
| Explicit type cast in `sql/10_stg/*`. Use for `mysql datetime ->
  timestamptz (UTC)` and similar.

| `cast:<from>-><to>`
| Same, with the cast directions documented inline (rare; only when
  the default cast is wrong).

| `id_map`
| Resolved through `migration._id_map_<source_table>` to a UUID. See
  xref:reference-id-maps.adoc[id maps].

| `crosswalk:<rfrnc>`
| Mapped through `mysql_raw.crosswalk_<rfrnc>` (loaded from
  `reports/crosswalks/<rfrnc>.csv`) to a DEMOS text ID. See
  xref:../sme/howto-author-a-crosswalk-csv.adoc[author a crosswalk
  CSV].

| `tag_pivot:<tag_name>`
| Pivoted out of an `mdcd_*_pgm_dtl` source table into a row in
  `demos_app.demonstration_type_tag_assignment` (resolving `<tag_name>`
  to its `tag_name_id`), driven by `reports/pgm_dtl_tag_mapping.csv`.
  The `20_app` transform that emits these rows is not yet authored. See
  xref:../sme/reference-pgm-dtl-tag-mapping.adoc[the tag mapping].

| `json_pack:<field>`
| Carried into a JSONB column under the named field (e.g.
  `validation_data`). See
  xref:reference-jsonb-schema-registry.adoc[the JSONB schema
  registry].

| `json_extract:<path>`
| Extracted from a JSONB column at the named path back into a flat
  column.

| `filter_typed:<predicate>`
| Source column rows are filtered into a typed child table
  (e.g. `application_date` rows where `date_type='effective_date'`).

| `derive`
| Computed in SQL from other columns; no 1:1 source.

| `pending_unify`
| Combined with the matching `mdcd_pendg_*` row per the
  xref:../sme/reference-pending-approved-rules.adoc[pending/approved
  rules].

| `comment_route`
| Routed to `private_comment` or `public_comment` per
  xref:../sme/explanation-comments-routing.adoc[comments routing].
  The same source column appears twice in the CSV (once per
  destination), differentiated by `notes`.

| `drop`
| Source column not migrated. Source table must also be on the drop
  list.
|===

== Match-marker semantics

The generator emits one of `OK`, `WARN`, `MISMATCH`, or `DROP` per
row. The cutoff is the normalised type comparison and whether the
transform "explains" the diff:

* `OK` and `DROP` are noise-free.
* `WARN` is the expected state for most rows that go through a
  semantic transform (`id_map`, `crosswalk:*`, `tag_pivot:*`,
  `json_pack:*`).
* `MISMATCH` is unreviewed -- treat it as a TODO.

== Editing workflow

. Edit `reports/source_target_columns.csv` (one row per column pair;
  multiple rows allowed when a source column splits into multiple
  DEMOS targets).
. Run `make column-map` from `docs/`. The generator runs lints
  before writing the fragments and exits non-zero on any lint
  failure.
. Run `make html` from `docs/`.

== Tests

* `tests/test_source_target_columns.py` -- golden-file render plus
  lint-failure cases. Run with `pytest tests/test_source_target_columns.py`.
