= Reference: derivability audit
:toc: left

Which DEMOS target columns do *not* come 1:1 from their mapped source table,
and of those, which can be _derived_ from other ("Nth") source tables versus
defaulted or supplied by a human? Useful when authoring a new `derive`,
`crosswalk`, or `tag_pivot` transform: this page is the live inventory of the
gaps those transforms must close. The body is generated by
`scripts/derivability_audit.py` from the pinned Prisma DDL, the schema
snapshot, and the `reports/inputs/derivability_verdicts.yaml` sidecar; run
`make derivability` in `docs/` to refresh it. The SME-facing view of the same
data is xref:../sme/explanation-derivability.adoc[the derivability explanation].
