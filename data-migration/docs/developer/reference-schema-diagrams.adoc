= Reference: schema diagrams (generated)

The entity-relationship diagrams below are *generated*, not hand-drawn.
`docs/tools/schema_diagrams_to_adoc.py` reads the authoritative target
schema -- the Prisma DDL (`state/prisma_ddl/<sha>.sql`, the concatenated
migration set) plus the captured FKs (`state/prisma_fks.json`) -- and
emits one diagram per table _class_ into `docs/shared/generated/`.
Regenerate with `make schema-diagrams` (in `docs/`); the partials are
committed and CI fails on drift.

`*_history` tables are excluded (they mirror their parent's columns).
Cardinality is derived from the DDL: a relationship is one-to-one when
the child's FK columns are its PK or a UNIQUE key, otherwise
zero-or-many; the parent side is optional (`|o`) when the FK column is
nullable. Each edge is labelled with its child-side FK column(s).

Tables are assigned to a class structurally (`*_limit` ->
type limiter; composite PK -> associative; single `text` id PK ->
static constraint; else data table), with curated exceptions in
`docs/tools/table_classes.yaml`. `make schema-drift` cross-checks the
generated model against the legacy hand-authored `DEMOS_Data_Model.mmd`
and writes `reports/schema_drift_report.md` (warn-only).

See xref:reference-prisma-ddl.adoc[the Prisma DDL pipeline] for how the
DDL artifact is fetched and pinned.
