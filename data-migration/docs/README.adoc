= DEMOS data migration: overview
:toc: left

This is the documentation set for the DEMOS data migration -- the
project that moves program data off the legacy PMDA system and onto the new
database that backs DEMOS. Go-live is planned for
*Wednesday, July 1, 2026*.

This page is the high-level briefing. Use the timeline below to see
where we are, then jump into the audience book that matches your role.

== At a glance

* *Goal.* Replace the legacy PMDA MySQL database with a new PostgreSQL database that
  backs the new DEMOS application, with no loss of meaningful data and no
  surprises for end users.
* *Approach.* Stand the new system up alongside the old one, check it
  against the SME-defined source of truth, rehearse the switch twice, and cut over
  in a planned window.
* *Go-live.* Wednesday, *2026-07-01*.
* *Where we are today (2026-07-09).* Mid-build: foundation is complete and the
  "move & reshape the data" phase is in flight. The first application-core
  loaders have landed (identity, demonstration, and the RBAC role assignments)
  while the deliverable / document / comment / amendment loaders remain
  outstanding. The go-live date is held at 2026-07-01; delivered work currently
  trails the calendar.

.What each phase delivers
[cols="1,3"]
|===
| Phase | Done when...

| Foundation & design
| The new database is structured and the major data-shape decisions
  are signed off.

| Move & reshape the data
| Every record from the legacy system has been carried over and
  reshaped so the new application can read it.

| Integration & user testing
| DEMOS runs against the new database and clears a full
  round of user-acceptance testing with no severe issues open.

| Cutover rehearsals
| The full switch-over procedure has been timed end-to-end, twice,
  with no surprises and a validated rollback.

| Go-live & hypercare
| The new system is in production, supported through a four-day
  hypercare window, with legacy PMDA queued for retirement.
|===
