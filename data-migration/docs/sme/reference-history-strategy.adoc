= Reference: history strategy

Live contents of `reports/narrative/history_strategy.md`. The short version: this
migration does **not** populate the DEMOS `*_history` tables. They are
owned by DEMOS and left empty at cutover for the `log_changes_*` capture
triggers to fill afterwards.

include::../shared/generated/live/history-strategy.adoc[]

== Why we do not backfill

See xref:../developer/explanation-history-backfill.adoc[the developer
explanation: history is DEMOS-owned, not migrated].

Per the SMEs, the DEMOS history tables do not have to be populated; DEMOS
triggers history-row creation on its own once the capture triggers are
installed (after constraints, via `refreshDbObjects.ts`). Every
`demos_app.*_history` table is therefore left EMPTY by this migration --
including the entities that previously had a MySQL `*_hstry` shadow.
