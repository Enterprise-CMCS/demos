== Highlights

* *Pending and approved fold into one application.* Status flow
  captures the lifecycle. See
  xref:howto-review-pending-approved-decisions.adoc[pending/approved
  review].
* *Program-detail tables collapse into tag assignments.* ~70 approved
  source tables (plus their `mdcd_pendg_*` counterparts) become rows
  in `demonstration_type_tag_assignment` keyed
  by `tag_name`. See
  xref:reference-pgm-dtl-tag-mapping.adoc[tag mapping].
* *Budget-neutrality machinery becomes one workbook row + JSONB.*
  See xref:howto-review-bn-jsonb-payload.adoc[review a BN payload].
* *All ~10 document tables collapse into one `document` table* with
  `document_type_id` as the discriminator.
* *Comments split by origin* into `private_comment` (CMS) or
  `public_comment` (state). See
  xref:explanation-comments-routing.adoc[comments routing].
* *Reference tables become text-ID lookups.* See
  xref:reference-static-constraint-tables.adoc[the static-constraint
  reference] and
  xref:howto-author-a-crosswalk-csv.adoc[crosswalks].

== What is dropped

See xref:reference-drop-list.adoc[the drop list].

== What is net-new in DEMOS

* `tag*` -- the tag system itself.
* `application_tag_suggestion*` -- AI suggestion artifacts.
* `uipath_*` -- UiPath integration.
* `deliverable_extension*` -- formal extensions of deliverable due
  dates.
* `event` -- audit event stream.

These are empty at cutover and populated by the new app over time.
