== How it is consumed

The `20_app` tag-assignment transform (not yet authored) reads this
CSV. For every row in the mapping, it emits a
`demos_app.demonstration_type_tag_assignment` row keyed on the
`source_table` and resolved from `tag_name` to its `tag_name_id`.

`mdcd_pendg_*_pgm_dtl` (pending) tables are *not* listed here -- the
transform infers them from the corresponding approved table and the
pending/approved rule.
