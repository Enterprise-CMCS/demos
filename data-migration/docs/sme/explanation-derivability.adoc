= Derivability: can every target column be derived from source?

When a DEMOS column is not a straight copy of a mapped PMDA column, the
migration must _derive_ it, default it, or ask a human for it. This page is the
honest answer to "can we derive everything?" for the tables mapped so far: the
verdict per gap, the residue that no source table can supply, and how much of
the remaining work is still pending your sign-off. The verdict for each gap is
your call, recorded in `reports/inputs/derivability_verdicts.yaml`; everything else is
generated from the schema, so the numbers here never drift.
