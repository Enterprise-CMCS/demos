= Reference: source/target column map

Column-level join between the MySQL PMDA source schema and the
DEMOS PostgreSQL target schema. The
xref:reference-source-target-map.adoc[top-level source/target map]
is the table-arrow view; this page drills each arrow into per-column
rows so the SME can sign off the column-level decisions.

== Inputs

* MySQL PMDA source schema: the live information_schema snapshot at
  `reports/schema_snapshot/columns.csv` (from `migrate schema-snapshot`).
* DEMOS target schema: the pinned Prisma DDL
  (`reports/prisma_ddl.sha256` + `state/prisma_ddl/<sha>.sql`).
* Mapping CSV (SME-curated): `reports/source_target_columns.csv`.
* Drop list: `reports/narrative/drop_list.md`.

The fragments below are regenerated by
`docs/tools/source_target_columns_to_adoc.py` -- run `make column-map`
in `docs/` after editing the CSV. Both schemas come from their
authoritative source of truth (the snapshot and the pinned DDL), so
the column map never drifts from the real schemas.

== Match marker legend

[cols="1,5"]
|===
| Marker | Meaning

| `OK`
| MySQL and DEMOS types are the same after normalisation.

| `WARN`
| Types differ but the listed `transform` explains the difference
  (e.g. `int` -> `uuid` via `id_map`, `int` -> `text` via
  `crosswalk:<rfrnc>`, anything -> `jsonb` via `json_pack:<field>`).

| `MISMATCH`
| Types differ and the transform does not explain it. Either the
  transform is wrong or the types are wrong; please flag.

| `DROP`
| Source column is on the drop list and has no DEMOS target.
|===

== Review checklist

* [ ] Every MySQL table appears at least once (mapped or dropped).
  The coverage table reports `100.0%` when complete.
* [ ] Every `transform` matches the SME's intent (copy vs cast vs
  crosswalk vs id_map vs tag_pivot vs json_pack/json_extract vs
  filter_typed vs derive vs drop vs pending_unify vs comment_route).
* [ ] Every `MISMATCH` row is either retyped or has a transform
  added.
* [ ] Every `crosswalk:<X>` references a CSV that exists under
  `reports/crosswalks/`.
* [ ] Every `tag_pivot:<X>` references a row in
  xref:reference-pgm-dtl-tag-mapping.adoc[`pgm_dtl_tag_mapping.csv`].
* [ ] Renames and folds are explained in `notes`.
* [ ] No drop-list table appears in the mapping CSV with a non-`drop`
  transform (the build catches this).
