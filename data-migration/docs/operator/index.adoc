== Cutover state machine

[mermaid]
....
stateDiagram-v2
  [*] --> Live
  Live --> Freeze: planned start
  Freeze --> Delta: pgloader delta
  Delta --> Build: stg + app + history
  Build --> Constraints: FKs + triggers + indexes
  Constraints --> Parity: pure-PG parity checks
  Parity --> Flip: go decision
  Parity --> Rollback: no-go
  Flip --> Hypercare
  Hypercare --> Decom: stable for N days
  Rollback --> Live
  Decom --> [*]
....

For the gating philosophy and rollback triggers, see
xref:explanation-cutover-state-machine.adoc[the cutover state machine
explanation].
